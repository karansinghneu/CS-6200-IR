The gradient of the image is one of the fundamental building blocks in <a class="mw-redirect" href="/wiki/Image_processing" title="Image processing">image processing</a>. For example, the <a href="/wiki/Canny_edge_detector" title="Canny edge detector">Canny edge detector</a> uses image gradient for <a href="/wiki/Edge_detection" title="Edge detection">edge detection</a>. In <a href="/wiki/Graphics_software" title="Graphics software">graphics software</a> for <a class="mw-redirect" href="/wiki/Digital_image_editing" title="Digital image editing">digital image editing</a>, the term gradient or <a href="/wiki/Color_gradient" title="Color gradient">color gradient</a> is also used for a gradual blend of <a href="/wiki/Color" title="Color">color</a> which can be considered as an even <a class="extiw" href="https://en.wiktionary.org/wiki/gradation" title="wiktionary:gradation">gradation</a> from low to high values, as used from white to black in the images to the right. Another name for this is <i>color progression</i>.\n</p><p>Mathematically, the <a href="/wiki/Gradient" title="Gradient">gradient</a> of a two-variable function (here the image intensity function) at each image point is a 2D <a class="mw-redirect" href="/wiki/Vector_(geometric)" title="Vector (geometric)">vector</a> with the components given by the <a href="/wiki/Derivative" title="Derivative">derivatives</a> in the horizontal and vertical directions. At each image point, the gradient vector points in the direction of largest possible intensity increase, and the length of the gradient vector corresponds to the rate of change in that direction.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>\n</p><p>Since the intensity function of a digital image is only known at discrete points, derivatives of this function cannot be defined unless we assume that there is an underlying <a href="/wiki/Continuous_function" title="Continuous function">continuous</a> intensity function which has been sampled at the image points. With some additional assumptions, the derivative of the continuous intensity function can be computed as a function on the sampled intensity function, i.e., the digital image. Approximations of these derivative functions can be defined at varying degrees of accuracy. The most common way to approximate the image gradient is to <a href="/wiki/Convolution" title="Convolution">convolve</a> an image with a kernel, such as the <a href="/wiki/Sobel_operator" title="Sobel operator">Sobel operator</a> or <a href="/wiki/Prewitt_operator" title="Prewitt operator">Prewitt operator</a>.\n</p><p>Image gradients are often utilized in <a href="/wiki/Map" title="Map">maps</a> and other visual representations of <a href="/wiki/Data" title="Data">data</a> in order to convey additional information. <a href="/wiki/Geographic_information_system" title="Geographic information system">GIS</a> tools use color progressions to indicate <a href="/wiki/Elevation" title="Elevation">elevation</a> and <a href="/wiki/Population_density" title="Population density">population density</a>, among others.\n</p>\n<div class="toc" id="toc"><input class="toctogglecheckbox" id="toctogglecheckbox" role="button" style="display:none" type="checkbox"/><div class="toctitle" dir="ltr" Gray pixels have a small gradient; black or white pixels have a large gradient.</div></div></div>\n<p>Image gradients can be used to extract information from images. Gradient images are created from the original image (generally by convolving with a filter, one of  the simplest being the <a class="mw-redirect" href="/wiki/Sobel_filter" title="Sobel filter">Sobel filter</a>) for this purpose. Each pixel of a gradient image measures the change in intensity of that same point in the original image, in a given direction. To get the full range of direction, gradient images in the x and y directions are computed.\n</p><p>One of the most common uses is in edge detection. After gradient images have been computed, pixels with large gradient values become possible edge pixels. The pixels with the largest gradient values in the direction of the gradient become edge pixels, and edges may be traced in the direction perpendicular to the gradient direction. One example of an edge detection algorithm that uses gradients is the <a href="/wiki/Canny_edge_detector" title="Canny edge detector">Canny edge detector</a>.\n</p><p>Image gradients can also be used for robust feature and texture matching. Different lighting or camera properties can cause two images of the same scene to have drastically different pixel values. This can cause matching algorithms to fail to match very similar or identical features. One way to solve this is to compute texture or feature signatures based on gradient images computed from the original images. These gradients are less susceptible to lighting and camera changes, so matching errors are reduced.\n</p>\n<h2><span class="mw-headline" id="Math">Math</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Image_gradient&amp;action=edit&amp;section=2" title="Edit section: Math">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>The gradient of an image is a vector of its <a href="/wiki/Partial_derivative" title="Partial derivative">partials</a>:<sup class="reference" id="cite_ref-dip3_2-0"><a href="#cite_note-dip3-2">[2]</a></sup><sup class="reference" style="white-space:nowrap;">:<span>165</span></sup>\n</p>\n<dl><dd><span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\\displaystyle \\nabla f={\\begin{bmatrix}g_{x}\\\\g_{y}\\end{bmatrix}}={\\begin{bmatrix}{\\frac {\\partial f}{\\partial x}}\\\\{\\frac {\\partial f}{\\partial y}}\\end{bmatrix}}}" xmlns="http://www.w3.org/1998/Math/MathML">\n<semantics>\n<mrow class="MJX-TeXAtom-ORD">\n<mstyle displaystyle="true" scriptlevel="0">\n<mi mathvariant="normal">&#8711;<!-- &#8711;
